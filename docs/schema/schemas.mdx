# Schemas

A database contains one or more named schemas, which in turn contain tables. Within one schema, two tables cannot have
the same name. The same table name can be used in different schemas without conflict; for example, both schema1 and
schema2
can contain tables named mytable.

## Get schema

You can either get a single schema or a collection of schemas. The schema object contains all nested object like
tables, columns, indexes and constraints. Use the `namesOnly` parameter to omit nested objects.

<Tabs>
    <TabItem value="http" label="HTTP" default>

        ```http title="All schemas, names only"
        GET https://api.centia.io/api/v4/schemas?namesOnly HTTP/1.1
        Accept: application/json; charset=utf-8
        Authorization: Bearer abc123
        ```

        ```http title="Specific schemas"
        GET https://api.centia.io/api/v4/schemas/myschema,anotherschema HTTP/1.1
        Accept: application/json; charset=utf-8
        Authorization: Bearer abc123
        ```

    </TabItem>
    <TabItem value="cli" label="CLI">

        ```bash
        centia schema get "myschema,anotherschema"
        ```

    </TabItem>
</Tabs>

## Add schema

A new schema object can be empty (name is the only property) or contain nested tables.

<Tabs>
    <TabItem value="http" label="HTTP" default>

        ```http title="Create new empty schema"
        POST https://api.centia.io/api/v4/schemas HTTP/1.1
        Content-Type: application/json
        Authorization: Bearer abc123

        {
            "name": "myschema"
        }
        ```

    </TabItem>
    <TabItem value="cli" label="CLI">

        ```bash
        centia schema add "myschema"
        ```


    </TabItem>
</Tabs>

## Rename schema

<Tabs>
    <TabItem value="http" label="HTTP" default>
        ```http
        PATCH https://api.centia.io/api/v4/schemas/myschema HTTP/1.1
        Content-Type: application/json
        Authorization: Bearer abc123

        {
            "name": "myschema2"
        }
        ```
    </TabItem>
    <TabItem value="cli" label="CLI">
        ```bash
        centia schema rename "myschema" "myschema2"
        ```
    </TabItem>
</Tabs>

## Delete schema

Be careful when delete schemas - all nested objects will also be deleted!

<Tabs>
    <TabItem value="http" label="HTTP" default>
        ```http
        DELETE https://api.centia.io/api/v4/schemas/myschema HTTP/1.1
        Authorization: Bearer abc123
        ```
    </TabItem>
    <TabItem value="cli" label="CLI">
        ```bash
        centia schema drop "myschema"
        ```
    </TabItem>
</Tabs>
