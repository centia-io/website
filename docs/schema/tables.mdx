# Tables

A table is much like a table on paper: It consists of rows and columns. The number and order of
the columns is fixed, and each column has a name. The number of rows is variable â€” it reflects how much data is stored
at a given moment.

## Add table

A new table object can be empty (name is the only property) or contain nested columns, indexes and constraints.

<Tabs>
    <TabItem value="http" label="HTTP" default>

        ```http title="Create new empty table"
        POST https://api.centia.io/api/v4/schemas/rockhall HTTP/1.1
        Content-Type: application/json
        Authorization: Bearer abc123

        {
            "name": "inductees"
        }
        ```

    </TabItem>
    <TabItem value="cli" label="CLI">

        ```bash
        centia table add "rockhall" "inductees"
        ```


    </TabItem>
</Tabs>

## Get table

You can either get a single table or a collection of tables. The schema object contains all nested objects like
columns, indexes and constraints. Use the `namesOnly` parameter to omit nested objects.

<Tabs>
    <TabItem value="http" label="HTTP" default>

        ```http title="All tables, names only"
        GET https://api.centia.io/api/v4/schemas/rockhall/tables?namesOnly HTTP/1.1
        Accept: application/json; charset=utf-8
        Authorization: Bearer abc123
        ```

        ```http title="Specific tables"
        GET https://api.centia.io/api/v4/schemas/rockhall/tables/inductees,anothertable HTTP/1.1
        Accept: application/json; charset=utf-8
        Authorization: Bearer abc123
        ```

    </TabItem>
    <TabItem value="cli" label="CLI">

        ```bash
        centia table get "rockhall" "inductees"
        ```

    </TabItem>
</Tabs>

## Rename and move table

You can move a table to another schema by setting the `schema` property and renaming the table by setting the
`name` property.

<Tabs>
    <TabItem value="http" label="HTTP" default>
        ```http
        PATCH https://api.centia.io/api/v4/schemas/rockhall/tables/inductees HTTP/1.1
        Content-Type: application/json
        Authorization: Bearer abc123

        {
            "name": "members",
            "schema": "rock_roll_of_hall"
        }
        ```
    </TabItem>
    <TabItem value="cli" label="CLI">
        ```bash
        centia schema rename "rock_roll_of_hall" "inductees" "members"
        ```
        ```bash
        centia schema move "rock_roll_of_hall" "inductees" "members"
        ```
    </TabItem>
</Tabs>
